## Overview

A full-stack web application that detects whether an image was generated by AI or captured by a real camera, using an ensemble of four deep learning models trained on the GenImage benchmark.

**Key Results:**
| Model | Accuracy | F1 | AUC |
|-------|----------|----|-----|
| Baseline CNN | 78.49% | 0.7921 | 0.8761 |
| Wavelet CNN | 83.35% | 0.8335 | 0.9122 |
| EfficientNet-B3 | 90.87% | 0.9078 | 0.9686 |
| **ResNet-50** | **91.37%** | **0.9149** | **0.9680** |

---

## Features

- **4-Model Ensemble** — Baseline CNN, Haar Wavelet CNN, EfficientNet-B3, ResNet-50
- **6-Pass Test-Time Augmentation** — flip, brightness ×2, contrast, rotation for stable predictions
- **Grad-CAM Visualization** — spatial attention heatmaps showing what the model looks at
- **Reliability-Weighted Voting** — consensus weighted by each model's validation accuracy
- **AI Forensic Report** — Claude claude-sonnet-4-5 (Anthropic) generates expert-level XAI analysis
- **Real-Time Web Interface** — FastAPI backend + React frontend
- **HEIC Support** — accepts iPhone photos directly

## Setup & Running

### Prerequisites
- Python 3.10+
- Node.js 18+
- Model checkpoints (see below)

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/wisconsin-ai-detector.git
cd wisconsin-ai-detector
```

### 2. Download Model Checkpoints

The `.pt` checkpoint files are not stored in this repo due to file size.
Download them from Google Drive: **[LINK HERE]**

Place them in `backend/checkpoints/`:
```
backend/checkpoints/baseline_cnn_best.pt
backend/checkpoints/wavelet_cnn_best.pt
backend/checkpoints/efficientnet_b3_best.pt
backend/checkpoints/resnet50_best.pt
```

### 3. Backend Setup

```bash
cd backend
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
python3 -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Backend runs at: `http://localhost:8000`

### 4. Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at: `http://localhost:5173`

## Model Architectures

### Baseline CNN
Custom 4-layer CNN trained from scratch. ~442K parameters.
- Conv2D blocks: [3→32, 32→64, 64→128, 128→256]
- Adam lr=1e-3, ReduceLROnPlateau, 25 epochs

### Wavelet CNN
Extends Baseline CNN with 12-channel Haar DWT input.
- Single-level 2D DWT decomposes each RGB channel into 4 sub-bands (LL, LH, HL, HH)
- CosineAnnealingLR scheduler

### EfficientNet-B3
ImageNet pretrained, two-stage fine-tuning.
- Stage 1 (epochs 1-5): frozen backbone, lr=1e-3
- Stage 2 (epochs 6-25): full fine-tune, lr=1e-4

### ResNet-50
ImageNet pretrained, two-stage fine-tuning.
- Same strategy as EfficientNet-B3
- Grad-CAM targets layer4[-1]

### Ensemble Voting
Reliability-weighted consensus based on validation accuracy:
```
ResNet-50:       0.9137
EfficientNet-B3: 0.9087
Wavelet CNN:     0.8335
Baseline CNN:    0.7849
```

## Dataset

**GenImage Benchmark** — 27,986 images
- Real: MS COCO (12,000 training images)
- AI: 7 generator families × 1,714 images each
  - BigGAN, VQDM, Stable Diffusion v5, Wukong, ADM, GLIDE, Midjourney
- Split: 23,998 train / 1,994 val / 1,994 test
- Balanced 50:50 real/AI in all splits

## Generalization Results (Phase 5)

Tested on unseen generators not in training:

| Source | Accuracy | Notes |
|--------|----------|-------|
| GenImage test set | 91.37% | In-distribution |
| DALL-E 3 (n=30) | 63.3% | Unseen generator |
| iPhone photos (n=29) | 48.3% | Computational photography |

**Generalization gap: -35.4pp** — driven by DALL-E 3's newer architecture and iPhone's Deep Fusion/HDR processing which produces unnaturally smooth images.

## Robustness Testing

ResNet-50 under image degradation:

| Condition | Max Drop | Notes |
|-----------|----------|-------|
| Gaussian Blur | -40.4pp | Destroys high-freq features |
| Gaussian Noise | -22.4pp | Masks artifact signals |
| JPEG Compression | -17.9pp | Attenuates texture |
| Brightness Shift | -4.4pp | Largely invariant |

## XAI — Forensic Report

The Forensic Report tab uses Claude claude-sonnet-4-5 (Anthropic) to generate a structured expert analysis:

1. **Verdict Summary** — single conclusion with confidence level
2. **Evidence Breakdown** — per-model analysis weighted by reliability
3. **Confidence Analysis** — P(Real) vs P(AI) spread interpretation
4. **Feature Detection** — visual artifacts driving the classification
5. **Reliability Assessment** — how much to trust the verdict
6. **Devil's Advocate** — strongest case for the opposing verdict

Requires an Anthropic API key (entered in the UI, never stored).
